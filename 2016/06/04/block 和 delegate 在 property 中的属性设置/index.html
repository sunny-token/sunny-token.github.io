block 和 delegate 在 property 中的属性设置

block

block在property中的属性应该是strong还是copy呢？

答案是:copy.

因为当block存储在property,NSArray或者其他数据结构中的时候，copy和strong是有很大的区别的,简单来说,你应该使用copy.当block第一次被创建的时候,它是被创建在栈上的,如果你调用block的时候,它已经被释放了,那么你就会得到一个很严重的bug < EXC_BAD_ACCESS>.
如果你用了strong(相当于retain)去创建block,其实它什么都没发生,它还是在栈上的.
如果你使用了copy属性去创建block,它将会被创建在堆上,那么你去使用的话,就不会出现 crash .并且如果你要把block保存在NSMutableArrays中，请一定要使用copy而不是strong或retain.

delegate

哪个好呢？
答案:weak.
因为:一般来说,两者的作用是一致的.
但是经过测试,声明一个delegate之前一直使用的是assign，今天调试一段代码的时候，发现程序会crash掉，报错EXC_BAD_ACCESS.

@property (nonatomic, assign) id delegate;
声明一个delegate，那么即便delegate指向的对象销毁了，delegate中依然会保存之前对象的地址
即，delegate成为了一个野指针…
@property (nonatomic, weak) id delegate;
而使用weak，则不会有上述问题，当delegate指向的对象销毁后，delegate = nil，
所以答案就是，使用weak。
